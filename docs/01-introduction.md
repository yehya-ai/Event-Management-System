# Introduction

## Problem Statement

Managing events manually using tools such as Excel sheets or messaging applications often leads to errors, poor organization, and data loss.  
It becomes difficult to track participants, manage event details, and ensure accurate and efficient coordination between organizers and attendees.

---

## Project Objectives

- Simplify the process of managing events and workshops  
- Reduce manual work and minimize human errors  
- Provide a centralized system for event registration  
- Improve organization and accessibility of event data  
- Enable administrators to track participants easily  

---

## Scope

### In Scope
- Creating events (add, edit, delete)  
- Viewing available events  
- Registering participants  
- Storing participant data  
- Admin dashboard for monitoring registrations  

### Out of Scope
- Online payment integration  
- Mobile application  
- Advanced authentication systems  
- External integrations  

---

## Assumptions

- Users have access to the internet  
- Admin has basic technical knowledge  
- The system will be used by a limited number of users initially  

---

## Constraints

- Limited development time  
- Small team size  
- Initial version will be basic (prototype)
