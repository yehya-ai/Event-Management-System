# Stakeholder Analysis

## Overview

Stakeholders are individuals or groups who have an interest in the system or are affected by it. Identifying stakeholders helps ensure that the system meets user needs and expectations.

---

## Stakeholders Identification

| Stakeholder        | Description |
|--------------------|------------|
| Admin              | Manages events and participants |
| Participant        | Registers and attends events |
| Development Team   | Builds and maintains the system |
| Instructor         | Reviews and evaluates the project |

---

## Stakeholder Analysis

### 1. Admin
- Role: System manager  
- Needs: Easy event management, participant tracking  
- Importance: High  

### 2. Participant
- Role: User of the system  
- Needs: Simple registration and browsing  
- Importance: High  

### 3. Development Team
- Role: System developers  
- Needs: Clear requirements and structure  
- Importance: Medium  

### 4. Instructor
- Role: Project supervisor  
- Needs: Organized work and proper implementation  
- Importance: High  

---

## Power vs Interest Matrix

| Stakeholder      | Power | Interest | Category |
|------------------|------|----------|----------|
| Admin            | High | High     | Manage Closely |
| Participant      | Low  | High     | Keep Informed |
| Development Team | Medium | Medium | Monitor |

## Conclusion

Understanding stakeholders helps in building a system that satisfies user requirements and ensures project success.
